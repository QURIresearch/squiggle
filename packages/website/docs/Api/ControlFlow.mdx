This page documents control flow. Squiggle has with if/else statements. Use reduce/map constructs instead of for loops.

## If-else

### If-else

```js
if condition then result else alternative
```

<SquiggleEditor
  defaultCode={`
if x == 1 then 1 else 2
`}
/>

### Ternary Operations

```js
test ? result : alternative
```

<SquiggleEditor
  defaultCode={`
x = 10
x == 0 ? 1 : 2
`}
/>

### Tips and tricks

#### Use brackets and parenthesis to simplify control flow

<SquiggleEditor
  defaultCode={`
if x == 1 then {
  1
} else {
  2
}
`}
/>

or

<SquiggleEditor
  defaultCode={`
if x == 1 then {
  (
    if y == 0 then {
      1
    } else {
      2
    }
  )
} else {
  3
}
`}
/>

#### Save the result to a variable

Assigning a value inside an if/else flow isn't possible:

```js
if x == 1 then {
  y = 1
} else {
  y = 2 * x
}
```

Instead, you can do this:

<SquiggleEditor
  defaultCode={`
y = if x == 1 then {
  1
} else {
  2 * x
}
`}
/>

Likewise, for assigning more than one value, you can't do this:

```js
y = 0
z = 0
if x == 1 then {
  y = 2
} else {
  z = 4
}
```

Instead, do:

<SquiggleEditor
  defaultCode={`
result = if x == 1 then {
  {y: 2, z: 0}
} else {
  {y: 0, z: 4}
}
y = result.y
z = result.z
`}
/>

## For loops

For loops aren't currently supported in Squiggle. Instead, use a [map](/docs/Api/List#map) or a [reduce](/docs/Api/List#reduce) operator.

Instead of:

```js
xs = [];
for (i = 0; i < 10; i++) {
  xs[i] = f(x)
}
```

do:

<SquiggleEditor
  defaultCode={`
ls = List.upTo(0,10)
xs = List.map(l, {|l| f(l)})
`}
/>
